# Claude Code Project Guidelines - MANDATORY ENFORCEMENT

## CRITICAL: Pre-Generation Protocol

**BEFORE generating or modifying ANY code, you MUST:**
1. **Read and acknowledge these rules**
2. **Plan how to stay within 80-character limit**
3. **Verify variable naming will follow m_/s_ conventions**

## MANDATORY Formatting Standards - NO EXCEPTIONS

### Line Width: ABSOLUTE 80-CHARACTER LIMIT
- **ENFORCE 80-character maximum for ALL files:**
  - C++ files (`c_src/*.cpp`, `c_src/*.hpp`)
  - Erlang files (`src/*.erl`)
  - Configuration files
  - Documentation files
- **NO EXCEPTIONS** for comments, strings, or function signatures
- **EXCEPTIONS** allow larger limit for tables, URLs, and embedded code
  examples in comments
- **BREAK ANY LINE** that exceeds 80 characters immediately
- **COUNT CHARACTERS** if you're unsure about line length

### Variable Naming: REQUIRED PREFIXES
- **Non-static class members**: MUST use `m_` prefix
- **Static class members**: MUST use `s_` prefix
- **NO EXCEPTIONS** - every class member variable requires correct prefix

### Indentation and Alignment
- **ALWAYS use 2-space indentation**
- **MAINTAIN columnar alignment** whenever possible
  ```cpp
  if (argc != 4                       ||
      !get(env,   argv[0], ctx)       ||
      !get(env,   argv[1], stripe_id) ||
      !get(env2,  argv[2], raw_fd)    ||
      !get(env31, argv[3], owner_pid))
    return enif_make_badarg(env);

  for (auto j = 0u; j < 64; ++j) {
    stripe->slots[j].fd        = -1;
    stripe->slots[j].stripe_id =  i;
    stripe->slots[j].slot_id   =  j;
  }

  struct FIFORequest {
    ErlNifPid             requester_pid{};      // Process waiting for reply
    std::atomic<uint64_t> request_timestamp{0}; // When request was sent (ns)
    std::atomic<uint32_t> timeout_ms{5000};     // Request timeout in msec
    std::atomic<bool>     active{false};        // Whether request is active
  };

  enum FIFOSlotStatus : uint32_t {
    // Start from 100 to avoid conflicts with existing SlotStatus enum
    SLOT_FIFO_RESERVED     = 100,  // Mode 3: Connection reserved for request
    SLOT_FIFO_REQUEST_SENT = 1001, // Mode 3: Request sent, awaiting reply
    SLOT_FIFO_DRAINING     = 10002 // Mode 3: Processing reply, about to release
  };
  ```
- **ALIGN wrapped function parameters** consistently

### Conditional Statements
- **Single-statement if/else**: NEVER use braces
  ```cpp
  // CORRECT
  if (condition)
    return true;

  // WRONG - don't generate this
  if (condition) {
    return true;
  }
  ```
- **Single-statement if with single-statement else**: prefer ternery `if` notation
  ```cpp
  // CORRECT
  return condition ? true_value : false_value;

  // WRONG - don't generate this
  if (condition)
    return true;
  else
    return false;

  // WRONG - don't generate this
  if (condition)
    return true;
  return false;
  ```

## Code Generation Protocol - FOLLOW EXACTLY

### Step 1: Before Writing Code
1. Check existing code style in the target file
2. Identify potential long lines that need breaking
3. Plan variable names with correct prefixes

### Step 2: During Code Generation
1. Count characters as you write each line
2. Break lines BEFORE they reach 80 characters
3. Apply m_/s_ prefixes to all class variables
4. Format single-line conditionals without braces

### Step 3: Post-Generation Verification - MANDATORY
**After generating ANY code, you MUST:**
1. **Scan every line** for 80-character violations
2. **Verify all class variables** have m_/s_ prefixes
3. **Check conditional formatting** follows single-line rules
4. **Report and fix ANY violations immediately**

### New atom initialization
**Prefer to declare and initialize atoms with NIFPP_ADD_KNOWN_ATOM
```cpp
NIFPP_ADD_KNOWN_ATOM(am_my_atom);
```

## Violation Handling - REQUIRED RESPONSE

**If you generate code that violates these standards:**
1. **IMMEDIATELY acknowledge the violation**
2. **Fix the violation without being asked**
3. **Explain what you changed and why**
4. **Re-verify the fixed code meets all standards**

## Validation Commands

**Run these after generating C++ code:**
```bash
# Check for 80-character violations (MUST return empty)
awk 'length($0) > 80 { print FILENAME ":" NR ": " $0 }' c_src/*.cpp c_src/*.hpp

# Verify no long lines exist
find c_src -name "*.cpp" -o -name "*.hpp" | xargs wc -L | awk '$1 > 80'
```

**If these commands show violations, you MUST check if the violations are not in URLs, tables or code examples in comments, and fix them before considering the task complete.**
