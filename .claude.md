# Claude Code Project Guidelines

## Formatting Standards

- **Line Width**: Maintain 80-character line width for all code files
  - C++ files (`c_src/*.cpp`, `c_src/*.hpp`)
  - Erlang files (`src/*.erl`)
  - Configuration files
  - Documentation files (`README.md`, etc.)
- **Indentation**: Keep 2 space indentation
- **Comments**: Keep inline comments within the 80-character limit

## Code Style

This project follows traditional C++ and Erlang formatting conventions
with strict line length limits for readability and consistency.

When writing or reformatting code, please:
1. Break long function signatures across multiple lines
2. Break long function calls with proper alignment
3. Keep string literals and comments within bounds
4. Use consistent indentation for wrapped lines
5. Name non-static local class variables with "m_" prefix
6. Name static local class variables with "s_" prefix
7. Whenever possible keep code columnar-aligned
8. If the body of an "if" statement has only one clause, don't wrap it in `{}`.
   Example:
   ```c++
   if (condition)
     return true;
   ```
